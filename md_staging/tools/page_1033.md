# Essential Tools for Windows Forms

## Overview

- Describes how to set and retrieve the card name for a child component in a CardLayout control.
- Demonstrates the use of methods related to card management.
- Highlighted methods include `GetCardName`, `GetCardNames`, `GetComponentFromName`, `GetNewCardName`, and `SetCardName`.

## Content

### Setting the Card Name

- The code examples provided show how to set the card name for a child component using the `SetCardName` method.

```csharp
this.cardLayout1.SetCardName(this.label1, "Card1");
```

```vb
Me.cardLayout1.SetCardName(Me.label1, "Card1")
```

### Property Window

- The figure below illustrates setting the card name for a `Label` control in the properties window of the CardLayout.

![Setting the Card Name](figure_665.png)

### Methods Associated with Card Name Property

The table below lists the methods associated with managing card names for child components in a CardLayout.

| Method                 | Description                                                                 |
|------------------------|-----------------------------------------------------------------------------|
| GetCardName           | Returns the card name of a Child component.                              |
| GetCardNames          | Returns an array containing the card names as strings.                  |
| GetComponentFromName  | Returns an associated control given a card name.                        |
| GetNewCardName        | Generates a new unique name for the card that could be added to this CardLayout. |
| SetCardName           | Sets the card name for a Child component.                               |

### Note

The card name property is added as an extended property in the properties window of the Child control added to the CardLayout.

## API Reference

### SetCardName

- **Description**: Sets the card name for a Child component.
- **Parameters**:
  - `childComponent`: The child control for which the card name is being set.
  - `cardName`: The name of the card to be set.
- **Returns**: Void.

### GetCardName

- **Description**: Returns the card name of a Child component.
- **Parameters**:
  - `childComponent`: The child control whose card name is to be retrieved.
- **Returns**: A string representing the card name.

### GetCardNames

- **Description**: Returns an array containing the card names as strings.
- **Parameters**: None.
- **Returns**: An array of strings representing the card names.

### GetComponentFromName

- **Description**: Returns an associated control given a card name.
- **Parameters**:
  - `cardName`: The name of the card to search for.
- **Returns**: The corresponding child control for the specified card name.

### GetNewCardName

- **Description**: Generates a new unique name for the card that could be added to this CardLayout.
- **Parameters**: None.
- **Returns**: A unique string for the new card.
