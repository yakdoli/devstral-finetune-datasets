#### Node Customization via `BaseStyle`

The `BaseStyle` property allows users to override the `Standard Style` settings for specified nodes. This enables the display of custom images and styles for nodes.

#### Figure 1473: Node Style inherited from another Base Style

This section illustrates how a node inherits its style from another `BaseStyle`. The screenshot shows the tree structure where nodes are visually differentiated based on their inherited styles. The nodes under each `Parent` are displayed in alternating colored backgrounds to emphasize the style inheritance.

### Explanation

This feature of the `TreeViewAdv` control allows developers to define multiple styling rules and apply them dynamically to different nodes. By setting the `BaseStyle` property, users can ensure a visually consistent and customized user interface.

## API Reference

- **Property**: `BaseStyle`
  - **Type**: `BaseStyle`
  - **Description**: Specifies the base style for the node.
  - **Default**: `null`
  - **Usage**: Assigns a custom `BaseStyle` to override the standard style settings.

## Code Examples

```csharp
// Example: Setting BaseStyle for a node
TreeAdvNode node = new TreeAdvNode("Node1");
node.BaseStyle = "BaseStyle1";
```

```vb
' Example: Setting BaseStyle for a node
Dim node As New TreeAdvNode("Node1")
node.BaseStyle = "BaseStyle1"
