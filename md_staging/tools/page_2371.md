# Essential Tools for Windows Forms

## Overview
- Explains how to manage pages in a control through the context menu.
- Describes the functionality of "Add Page" and how to edit the page settings via the XPTaskPage Collection Editor.
- Details the process of removing pages using the "Remove Page" option.

## Content

### Managing Pages in a Control

#### Through Context Menu of Header

**Figure 1529: Through Context Menu of Header**

The context menu from the header of the control provides several useful commands for managing pages, as shown in the figure. Here are some key points:

- **"Add Page" command** in the Property grid allows you to add a new page to the control.
- **XPTaskPage Collection Editor** can be accessed by selecting the "TaskPages" property of the control. This editor enables detailed page management, including the ability to remove pages using the "Remove page" option.

### Detailed Steps

1. **Adding a Page**:
   - Right-click the header of the control.
   - Select the "Add Page" command from the context menu.

2. **Editing Page Settings**:
   - Right-click the header or within the control area to access the context menu.
   - Select the "TaskPages" property to open the XPTaskPage Collection Editor.
   - Use the editor to manage page properties, including adding, removing, and reordering pages.

3. **Removing a Page**:
   - In the XPTaskPage Collection Editor, select the page you wish to remove.
   - Click the "Remove page" option to delete the selected page.

### Tips and Notes
- Always ensure that you save your changes after modifying page settings.
- Use the "Previous Page" and "Next Page" options to navigate through multiple pages efficiently.
- "Lock Controls" can be used to prevent accidental movement or resizing of controls within the page.

## API Reference

### Properties and Methods
- **TaskPages**: Accesses the XPTaskPage Collection Editor for managing pages.
- **Add Page**: Command to add a new page to the control.

### Events
- None explicitly mentioned in the context menu.
