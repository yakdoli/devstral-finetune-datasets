# Essential Tools for Windows Forms

## Overview

- Describes key input handling in Windows Forms using `PreviewKeyDownEventArgs`.
- Covers various members to check and manipulate key events and their modifiers.
- Includes example code demonstrating the usage of key event handling in C#.

## Content

### Key Event Handling in Windows Forms

#### Members and Description

| **Members**       | **Description**                                                                 |
|-------------------|---------------------------------------------------------------------------------|
| **Alt**           | Gets a value indicating whether the ALT key was pressed.                      |
| **Control**       | Gets a value indicating whether the CTRL key was pressed.                    |
| **IsInputKey**    | Gets/sets a value indicating whether a key is a regular input key.            |
| **KeyCode**       | Gets the keyboard code for a `System.Windows.Forms.Controls.KeyUp` / `System.Windows.Forms.Controls.KeyDown` event. |
| **KeyData**       | Gets the key data code for a `System.Windows.Forms.Controls.KeyUp` / `System.Windows.Forms.Controls.KeyDown` event. |
| **KeyValue**      | Gets the keyboard value for a `System.Windows.Forms.Controls.KeyUp` / `System.Windows.Forms.Controls.KeyDown` event. |
| **Modifiers**     | Gets the modifiers flag for a `System.Windows.Forms.Controls.KeyUp` / `System.Windows.Forms.Controls.KeyDown` event. |
| **Shift**         | Gets a value indicating whether the SHIFT key was pressed.                    |

## Code Examples

The example above demonstrates how to handle the `PreviewKeyDown` event in a `TabControlAdv` control. This event is triggered before a key is pressed and provides information about the key being pressed, including its code, value, and modifiers.

### Explanation of Code

- **Sender**: The `object` representing the control that raised the event.
- **e**: The `PreviewKeyDownEventArgs` object containing key-related information.
- **Console.Write**: Outputs messages to the console for debugging and logging purposes.

This example effectively demonstrates how to retrieve and log key event data at runtime, enabling developers to react to specific key inputs in their Windows Forms applications.
