# Essential Tools for Windows Forms

## Overview
- Demonstrates how to customize and persist toolbar states at runtime in Windows Forms using Syncfusion controls.
- Explains the use of AutoLoadToolBarPositions property for restoring toolbar positions and layouts.
- Provides an example of removing a "Bar Item" from the toolbar.

## Content

### Toolbar State Persistence
Figure 848 shows the properties window that allows customization of toolbar behavior.

- **AutoLoadtoolBarPositions**: Setting this property to true ensures that the toolbar positions are automatically loaded when the application restarts.
- **Other Properties**:
  - AutoLoadCustomData
  - AutoSaveCustomData
  - IgnoreWorkingArea
  - PopupCloseTimer
  - ShowDropShadow
  - ShowHighlightRectangle
  - ShowShadow

### Customizing the Toolbar at Runtime
We can customize the toolbar at runtime, and the changes can be persisted by setting the `AutoLoadToolBarPositions` property to true. The following screenshot (Figure 849) shows the Toolbar customization where the "Open" Bar Item is removed.

#### Figure 849: Removing the "Open" Bar Item

In this example, the "Open" Bar Item is explicitly removed from the toolbar.

## API Reference
- **Syncfusion.Windows.Forms.MainFrameBarManager**
  - **AutoLoadToolBarPositions**: A property that specifies whether the toolbar positions should be automatically loaded when the application is restarted.
  - **Other Relevant Properties**:
    - AutoLoadCustomData
    - AutoSaveCustomData
    - IgnoreWorkingArea
    - PopupCloseTimer
    - ShowDropShadow
    - ShowHighlightRectangle
    - ShowShadow

## Code Examples
Customizing the toolbar and enabling persistence can be achieved through code as well.

```csharp
// Example: Enabling AutoLoadToolBarPositions
mainFrameBarManager1.AutoLoadToolBarPositions = true;
```
