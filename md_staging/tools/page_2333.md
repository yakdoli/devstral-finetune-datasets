# Essential Tools for Windows Forms

## Overview
- Introduces reordering options (`Bring To Front`, `Send To Back`) in Designer for managing control layers.
- Explains the `WizardControlPage` properties `NextPage` and `PreviousPage` for specifying page order in Wizards.
- Demonstrates setting up page transitions using code examples.

## Content

### Registration Wizard Designer Interface
The image illustrates the `Registration Wizard` window in the Windows Forms Designer, showcasing a variety of options for managing controls and page flows:
#### Figure 1504: Accessing reordering Options(Bring To Front, Send To Back) in Designer

- **Highlighted Features:**
  - `Bring To Front` and `Send To Back` options allow rearranging control layers.
  - `WizardControlPage` properties determine the order of pages during runtime.

### Page Navigation using WizardControlPage Properties
- The `WizardControlPage` has the `NextPage` and `PreviousPage` properties:
  - These properties specify the order of page selection.
  - Users can set these properties in the designer for all pages.
  - If set, the `WizardControl` uses them as cues to determine the new page to be selected at runtime.

#### Table: `WizardControlPage` Properties
| WizardControlPage Property | Description                          |
|---------------------------|----------------------------------------|
| NextPage                 | Sets the next page of the wizard.     |
| PreviousPage             | Sets the previous page of the wizard. |

### Code Example for Setting Up Page Transitions
```csharp
this.wizardControlPage2.NextPage = this.wizardControlPage3;
```

## API Reference

### `WizardControlPage`

#### Properties
- **NextPage**:

  - Description: Specifies the next page to navigate to in the wizard.
- **PreviousPage**:
  - Type: `WizardControlPage`
  - Description: Specifies the previous page to navigate to in the wizard.
