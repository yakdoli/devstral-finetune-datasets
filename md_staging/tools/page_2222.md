# Essential Tools for Windows Forms

## Overview

- This section explains how to set a common style for all children of a parent node in a TreeViewAdv control.
- It demonstrates the use of the `ChildStyle` property to modify the appearance of all children under a specific parent node.
- The example focuses on changing the font for all child nodes of a parent node.

## Content

### Setting a Common Style for All the Children of a Parent Node

This topic illustrates how the appearance of all the children of a particular node can be affected by modifying the `ChildStyle` property of that `TreeNodeAdv`. In this example, we will modify the font for all the children of a parent node.

In the Node Collection Editor, modify the settings in the `TreeNodeAdv.ChildStyle` property. These changes will be reflected in the child node of this node.

#### Figure 1474: ChildStyle set using Node Collection Editor

![Node Collection Editor Screenshot](https://usercontent.blob.core.windows.net/images/pages/2222/TreeNodeAdv_NodeCollection_Editor.png)

The screenshot shows the Node Collection Editor where you can modify the appearance settings for the children of a selected parent node. The settings include:

- **Font**: Specifies the font type (e.g., Verdana, 8 pt).
- **Style**: Allows you to set the font style (e.g., Bold).
- **Background**: Sets the background style.
- **BaseStyle**: Defines the base style.
- **CheckButtonImage**: Configures the appearance of check button images.
- **CheckState**: Manages the check state (checked or unchecked).
- **Comparison**: Sets comparison properties for nodes.
- **Culture**: Defines the culture settings.
- **Enabled**: Controls the enabled state of nodes.
- **EnsureDefault**: Ensures default styles are applied.

By modifying these settings, you can customize the appearance of all child nodes under the selected parent node.

## API Reference

### Namespace: Syncfusion.Windows.Forms.Tools

#### Class: TreeNodeAdv
- **ChildStyle**: Property used to set the style for all child nodes under the parent node. This property allows customization of font, style, background, and other appearance settings.

## Code Examples

Here is an example of how to programmatically set the `ChildStyle` for all children of a parent node in C#:

```csharp
using Syncfusion.Windows.Forms.Tools;

// Assuming treeViewAdv is your TreeViewAdv instance
TreeNodeAdv parentNode = treeViewAdv.Nodes.Add("Parent Node");

// Set the ChildStyle for all children of parentNode
parentNode.ChildStyle.Font = new Font("Verdana", 8, FontStyle.Bold);
parentNode.ChildStyle.ForeColor = Color.Blue;

// Add child nodes
TreeNodeAdv childNode1 = parentNode.Nodes.Add("Child Node 1");
TreeNodeAdv childNode2 = parentNode.Nodes.Add("Child Node 2");
```

In the above example, the font of all child nodes under `parentNode` is set to Verdana, 8 pt, bold, and the foreground color is blue.
