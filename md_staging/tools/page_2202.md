# Essential Tools for Windows Forms

## Overview
- Displays scroll tips for scrollbars.
- Provides Design-Time Features for TreeView control.
- Accesses TaskWindow using smart tags.

## Content

### 3.21.3.4 DesignTime Features

TreeView provides the shortcuts to access various properties of the control.

**TaskWindow** can be accessed using smart tag.

![](url_for_image)

**Figure:** TreeViewAdv Tasks window in the Design-time environment

### Key Features:
- **Edit Nodes Collection**: Allows modification of the node structure of the TreeView control.
- **Appearance**:
  - Show Check Boxes
  - Show Option Buttons
  - Show Lines
  - Show Plus Minus
- **ImageList**:
  - Left ImageList
  - Right ImageList
  - State ImageList
- **Behavior**:
  - Edit Label
  - Load Child nodes OnDemand
  - Owner Draw Nodes
  - HotTracking
- **Layout**:
  - Anchor
  - Dock

## API Reference

### Properties
- **Name**: The name of the TreeViewAdv control (`treeViewAdv1`).
- **Show Check Boxes**: Boolean property to enable/disabled check boxes.
- **Show Option Buttons**: Boolean property to enable/disabled option buttons.
- **Show Lines**: Boolean property to draw lines between nodes.
- **Show Plus Minus**: Boolean property to show the expand/collapse icons.
- **Left ImageList**: Image list for the left icons in nodes.
- **Right ImageList**: Image list for the right icons in nodes.
- **State ImageList**: Image list for the state-specific icons in nodes.
- **Edit Label**: Boolean property to enable label editing.
- **Load Child nodes OnDemand**: Boolean property to load child nodes on demand.
- **Owner Draw Nodes**: Boolean property to allow custom drawing of nodes.
- **HotTracking**: Boolean property to enable hot tracking behavior.
- **Anchor**: Defines how the control is anchored to its parent control (`Top, Left`).
- **Dock**: Specifies how the control is docked to its parent (`None`).
