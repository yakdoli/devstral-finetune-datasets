### Toolbar Persistence

Close the application and again run the same application. The removed MenuBar item "Open" is sustained. The following screen shot illustrates this.

![Figure 850: "Open" Bar Item removed from the Toolbar](image.png)

#### Section: Save / Load the Bar Information

**3.7.4.9.1 Save / Load the Bar Information**

Toolbar Persistence can be done by using the below given properties and methods.

| **MainFrameBarManager Property**        | **Description**                                                                                                                                                                                                 |
|------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| AutoLoadToolBarPositions                  | Indicates whether to automatically load the persisted toolbar positions when the application is restarted.                                                                                           |
| AutoPersistCustomization                  | Indicates whether to persist and load user-customized info from/to isolated storage.                                                                                                             |
| AutoSaveCustomData                       | Gets or Sets a value indicating whether to automatically save user's customization data (new Bars/BarItems added/removed at runtime), when customization target form is closed.                          |
| AutoLoadCustomData                       | Gets or Sets a value indicating whether to automatically load user's customization data (new Bars/BarItems added/removed at runtime), when customization target form is activating.                 |

## API Reference

### MainFrameBarManager Properties
- **AutoLoadToolBarPositions**: A property that controls automatic loading of persisted toolbar positions.
- **AutoPersistCustomization**: A property that manages the persistence and loading of user-customized information.
- **AutoSaveCustomData**: A property related to automatically saving user's customization data.
- **AutoLoadCustomData**: A property related to automatically loading user's customization data.

### Methods
While the provided text does not specify explicit methods, the properties above represent methods for configuring persistence behavior in the application.

## Code Examples

Example of configuring toolbar persistence in code:

```csharp
// Enabling automatic loading of toolbar positions
MainFrameBarManager1.AutoLoadToolBarPositions = true;

// Enabling automatic persistence of user customization
MainFrameBarManager1.AutoPersistCustomization = true;

// Enabling automatic saving of user customization data
MainFrameBarManager1.AutoSaveCustomData = true;

// Enabling automatic loading of user customization data
MainFrameBarManager1.AutoLoadCustomData = true;
```
