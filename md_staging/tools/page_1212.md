## Content

### Merge Behavior Based on MergeType

The merge behavior is governed by the `MergeType` property of both the MDI Container Item and the MDI Child Item. The following table outlines the merge behavior for various combinations of `MergeType` values.

| MDI Container Item's MergeType | MDI Child Item's MergeType | Merge Behavior |
|---------------------------------|----------------------------|----------------|
| Add | Add / MergeItems / Replace | No merging. |
| Add | Remove | Parent item stays visible; child item hidden. |
| MergeItems | MergeItems | If both are sub-menus (ParentBarItems), their children will be merged together; otherwise the child item will replace the parent item. |
| MergeItems | Replace | Child item will replace parent item. |
| MergeItems | Add | No merging. |
| MergeItems | Remove | Both items will be hidden. |
| Remove | Remove | Both items will be hidden. |
| Remove | Add / MergeItems / Replace | Child item will replace parent item. |
| Replace | Add | No merging. |
| Replace | MergeItems | Parent item, if any, will replace the child item. |
| Replace | Replace | Child item will replace the parent item. |
| Replace | Remove | Parent item stays visible; child item will be hidden. |

You can choose anyone of the following for MDI merging.

## API Reference

The `MergeType` property is used to control the merging behavior. The possible values for `MergeType` include:
- **Add**: No merging occurs.
- **MergeItems**: Merges children of both items if they are sub-menus.
- **Replace**: Replaces the parent item with the child item.
- **Remove**: Hides both items or the child item, depending on the parent's `MergeType`.

## Code Examples

Here is a sample code snippet demonstrating how to set the `MergeType` property for an MDI item:

```csharp
using Syncfusion.Windows.Forms.Tools;

// Assuming bar1 is an instance of a Bar control
bar1.MergeType = MergeType.Add;

// Assuming childItem is an instance of an MDI child item
childItem.MergeType = MergeType.Replace;
```
