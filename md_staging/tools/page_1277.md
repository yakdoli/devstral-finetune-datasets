# Essential Tools for Windows Forms

## Overview
- This page demonstrates the TreeViewAdv NodeCollection Editor in Windows Forms.
- Focuses on accessing Subitems collection properties through the dialog box.
- Provides a detailed look into the properties and functionalities available within the TreeViewAdv control.

## Content

### TreeViewAdv NodeCollection Editor Overview
The TreeViewAdv NodeCollection Editor is a tool used to manage and edit hierarchical data structures in Windows Forms applications. The editor allows users to add, remove, and modify nodes in a tree-like structure, providing extensive control over the behavior and presentation of the data.

#### Properties and Behavior
The properties panel on the right side of the editor allows users to configure various aspects of the tree nodes. Key sections include:

- **Behavior**: Control settings like checked state, default options, expansion, and interactivity features.
- **Data**: Manages custom controls, primitives, subitems, and tags for each node.
- **Sorting**: Configures sorting options, comparers, and sort order for the tree nodes.

#### Visual Representation
The left side of the editor displays the hierarchical structure of the nodes. An example structure is shown:

| Country | Product    | Sales (%) |
|---------|------------|-----------|
| USA     | Tools      | 85        |
|         | Diagram    | 69        |
| Russia  | Grid       | 65        |
| China   | Chart      | 78        |

This structure represents a tree where countries are the parent nodes, and products are child nodes associated with specific sales percentages.

#### Interaction Controls
- **Add Node**: Adds a new node to the tree structure.
- **Add Child**: Adds a child node to the selected parent node.
- **Remove**: Deletes the selected node from the tree.
- **Select added node**: Ensures automatic selection of newly added nodes.
- **Sort**: Organizes the nodes based on specified sorting criteria.
- **OK/Cancel**: Saves or discards changes made to the tree structure.

### Figure 860: Accessing the SubItems Collection Dialog Box
This figure illustrates how to access and manage the SubItems collection dialog box within the TreeViewAdv NodeCollection Editor. The SubItems property is crucial for adding supplementary information to each node, enhancing the functionality and flexibility of the tree structure.

## API Reference
- **Namespace**: `Syncfusion.Windows.Forms.Tools`
- **Class**: `TreeViewAdv`
- **Properties**:
  - **Nodes**: Represents the collection of nodes in the tree.
  - **Behavior**:
    - `Checked`: Indicates whether the node is checked.
    - `EnsureDefaultOptions`: Ensures default options are set.
    - `Expanded`: Indicates the expanded state of the node.
    - `InteractiveCheckBox`: Enables interactive checkboxes for nodes.
    - `Optioned`: Indicates whether the node is in an option mode.
  - **Data**:
    - `CustomControl`: Specifies any custom control associated with the node.
    - `Primitives`: Indicates the primitives associated with the node.
    - `SubItems`: Manages subitems for the node.
    - `Tag`: Stores additional information for the node.
  - **Sorting**:
    - `CompareOptions`: Defines comparison options for sorting.
    - `Comparer`: Specifies a custom comparer for sorting.
    - `SortOrder`: Indicates ascending or descending order.
    - `SortType`: Defines the type of sorting (e.g., text-based sorting).

## Code Examples
Here is a sample code snippet demonstrating how to interact with the TreeViewAdv NodeCollection Editor:

```csharp
// Adding a new node to the TreeViewAdv control
treeViewAdv1.Nodes.Add("USA");
treeViewAdv1.Nodes[0].Nodes.Add("Tools", "85");
treeViewAdv1.Nodes[0].Nodes.Add("Diagram", "69");

// Accessing SubItems collection
var rootNode = treeViewAdv1.Nodes[0];
rootNode.SubItems.Add("Additional Info", "Some Data");

// Sorting nodes
treeViewAdv1.Sort();
```
