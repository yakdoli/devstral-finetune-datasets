# Essential Grid for Windows Forms

## Overview
- Explains various properties that affect the behavior of the grid when records are changed or deleted.
- Describes different flags that control how the grid handles notifications and updates.
- Includes handling the `ListChanged` event to retrieve the `ListChangedType`.

## Content

### Table: Grid Behavior Properties

| Property                        | Description                                                                                                                                                 |
|---------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ShouldIgnoreReset               | Specifies whether `ListChangedType.Reset` notification should be ignored.                                                                                   |
| ShouldInvalidateCounters        | Specifies whether counters need to be marked dirty when `ListChanged` event is handled.                                                                     |
| ShouldInvalidateGroupSortOrder  | Specifies whether changes to the current record affect the sort order of the current group.                                                                 |
| ShouldInvalidateScreen          | Specifies whether the grid should repaint the record when changes to the record were made in the underlying data source and `ListChangedType.ItemChanged` notification was raised. |
| ShouldInvalidateSummaries       | Specifies whether summaries need to be marked dirty when `ListChanged` event is handled.                                                                   |
| ShouldReevaluateSortPosition    | Specifies whether changes to the current record affect the sort order of the current group.                                                                 |
| ShouldResetCurrentRecord        | Specifies whether the current record should be reset when `ItemChanged` notification is received for the current record.                                   |

### Handling the ListChanged Event

You can handle this event to retrieve the `ListChangedType`. The following code example illustrates this.

#### Code Example: C#

```csharp
private void gridGroupingControl1_SourceListListChanged(object sender, TableListChangedEventArgs e)
{
    Console.WriteLine("ListChangedType :" + e.ListChangedType.ToString());
}
```

#### Code Example: VB.NET

```vb
Private Sub gridGroupingControl1_SourceListListChanged(ByVal sender As Object, ByVal e As TableListChangedEventArgs)
    Console.WriteLine("ListChangedType :" & e.ListChangedType.ToString())
End Sub
```
