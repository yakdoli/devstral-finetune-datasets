### Display of Grouping Grid Content in Word Document

The figure below illustrates how the Grouping Grid control content can be converted into a Word document format. This showcases the hierarchical structure of parent-child-grandchild relationships, maintaining the layout and formatting consistent with the original grid.

#### Figure 454: Grouping Grid control content converted to Word Document

The Word document representation includes the following:

- **Header**: Syncfusion Inc.
- **Parent Table**: Includes columns for `parentID`, `ParentName`, and `ParentDec`.
  - Example rows:
    - `parentID: 0`, `ParentName: parentName0`, `ParentDec: parentDec0`
    - `parentID: 1`, `ParentName: parentName1`, `ParentDec: parentDec1`
    - `parentID: 2`, `ParentName: parentName2`, `ParentDec: parentDec2`
    - `parentID: 3`, `ParentName: parentName3`, `ParentDec: parentDec3`
    - `parentID: 4`, `ParentName: parentName4`, `ParentDec: parentDec4`

- **Child Tables**: Nested under each parent row, showing `childID` and `Name`.
  - Example rows for `parentID: 0`:
    - `childID: 0`, `Name: ChildName 0`
    - `childID: 5`, `Name: ChildName 5`
    - `childID: 10`, `Name: ChildName 10`
    - `childID: 15`, `Name: ChildName 15`

  - Example rows for `parentID: 1`:
    - `childID: 1`, `Name: ChildName 1`
    - `childID: 6`, `Name: ChildName 6`

- **Grandchild Table**: Further layers of hierarchy under the child rows, showing `GrandChildID` and `Name`.
  - Example rows for `childID: 6`:
    - `GrandChildID: 6`, `Name: GrandChildName 6`
    - `GrandChildID: 26`, `Name: GrandChildName 26`
    - `GrandChildID: 46`, `Name: GrandChildName 46`

- **Sibling Child Tables**: Additional child tables at the same level as the first set.
  - Example rows for `parentID: 2`:
    - `childID: 11`, `Name: ChildName 11`
    - `childID: 16`, `Name: ChildName 16`

This structured table representation ensures that the hierarchical relationships are clearly visible and maintainable in the Word document format.
